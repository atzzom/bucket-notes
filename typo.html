Introduction

The color property sets and modifies the text color, allowing only one value, that can be expressed in different formats, as rgb, rgba, HSL, HSLa, and hexadecimal values. Font-based propertiesback to top

Within the available typography properties, they are classified in two categories: font-based properties and text-based properties. They are easily identifiable because most of these will be preface with either font-* or text-*.

Adding color to textback to top

Typeface vs. font

The terms "typeface" and "font" are often interchanged, causing confusion. Here is a breakdown of exactly what each term means.

A typeface is what we see. It is the artistic impression of how text looks, feels, and reads.

A font is a file that contains a typeface. Using a font on a computer allows the computer to access the typeface.

One way to help clarify the difference between a typeface and a font is to compare them to a song and and MP3. A typeface is very similar to a song in that it is a work of art. It is created by an artist or artists and is open to public interpretation. A font, on the other hand, is very similar to an MP3 in that it is not the artistic impression itself, but only a method of delivering the artistic value.

Font family

The font-family property is used to declare which font -as well as which fallback or substitute fonts- should be used to display text. The value of the font-family property contains multiple font names, all comma separated. The first declared font (starting from the left), is the primary font choice. If this font is unavailable, then, the alternative declared fonts will follow after it in order of preference from left to right.

When declaring fonts, take into account that font names with more than one word must be declared wrapped in quotation marks. Additionally, the las font should be a keyword value, which will use the system default font for the specified type, most commonly either sans-serif or serif.

The following is an example of the declaration of the font-family property.

body {
font-family: Garamond, Georgia, Lucida, serif;
}

Typeface weights

Before using a numeric value, we need to check and see whether the typeface we are using comes in the weight we'd like to use. Attempting to use a weight that's not available for a given typeface will cause those styles to default to the closest value. For example, the Times New Roman typeface comes in two weights: normal (400), and bold (700). Attempting to use a weight of 900 will default the typeface to the closest related weight, 700 in this case.

This property makes reference to the thickness of the typeface, is used to make the text bolder or lighter, this property accepts both, keyword and numerical values.

Keyword values include normal, bold, bolder, lighter, and inherit. Of these keyword values, it is recommended to primarily use normal and bold to change text from normal to bold an vice versa. Rather than using the keyword values bolder or lighter, it's better to use a numeric value for more specific control. Here is an example of the use of the font-weight property using keyword values:

.daring {
font-weight: bold;
}

Line heightback to top

Line height or leading, is the distance between two lines of text, is declared using the line-height property. The line-height property accepts all general length values, like pixels, percentages, and em.

The best practice for legibility is to set the line-height to around one and a half times the font-size property value. This could be quickly accomplished by setting the leading/line-height to 150%, or just 1.5. The box-shadow property accepts an optional forth length value, before the color value, for the spread of the shadow. Lastly, this property also may include and optional inset value at the beginning of the value to place the shadow inside an element as opposed to outside the element.

This property allow to add one or multiple shadows to a text. The property generally takes four values, all listed one after the other from left to right. The first three values are lengths, and the last value is a color.

Each of the following values represents in the following order:
- 1st: horizontal shadow's offset.
- 2nd: vertical shadow's offset.
- 3rd: shadow's blur radius.
- 4th: shadow's color.

Using negative length values for the horizontal and vertical offsets allows us to move shadows toward the left and the top.

Embedding web fontsback to top

Off course, it is also possible to use fonts additional to the web-safe fonts, indeed, it is possible to upload an specific font to a server in order to be rendered when the page is visited, this capability has opened unlimited options to choose from. Embedding a web font looks like the following CSS:

@font-face {
font-family: "Lobster";
src: local("Lobster"), url("lobster.woff") format("woff");
}
body {
font-family: "Lobster", "Comic Sans", cursive;
}

From the previous example, it can be observed that when embedding a web font it is necessary to:
- Use the @font-face at-rule to identify the font's name, via the font-family property.
- The font's source (the path to the font file containing the chosen font), via the src property.
- From there is possible to use the font by including its name within any font-family property value.

Take into account that the ability to embed any font on a website is totally different to have legally right to do so. As artistic work, fonts have licenses in order to be used. Fortunately, the value of using new fonts on websites have been recognized. Companies like Typekit and Fontdeck, work off a subscription mode for licensing fonts, while others, like Google Fonts, license fonts for free.

Citations & quotesback to top

Writing requires sometimes citing different sources or quotations. In HTML exists different markups to semantically cover this cases, specifically the <cite>, <q>, and <blockquote> elements. In order to know which of these elements use, the follow principles can be followed:

<cite>: used to reference a resource, creative work, or author.
<q>: used for short, inline quotations.
<blockquote>: used for longer external quotations (paragraph-like quotations).

Citing a creative workback to top

In HTML this element is used specifically to cite a creative work, the <cite> element must include either the title of the work, the author's name, or a URL reference to the work. By default, content wrapped within the <cite> element will appear in italics. When relevant, is useful to include and additional reference/hyperlink to the original source of the citation.

Here the book Steve Jobs, by Walter Isaacson, is reference within the <cite> element. Inside the citation is also a hyperlink to the book.

<p>The book <cite><a href="http://www.amazon.com/Steve-Jobs-Walter-Isaacson/dp/1451648537">Steve Jobs</a></cite> is truly inspirational.
