Forms intro

Forms are used to retrieve information given by the user, process requests among others. Additionally, form gotten information can be used further by the user in a search query, a username or password. Here, the form main aspects are going to be explained, how do form elements are marked up, skipping the details about the back–end (server) interaction.

Initializing a form

To introduce a form the <form> element must be used, this identifies where on the page the form controls will appear, wrapping any child element within the form into an opening and closing tag, similarly to a <div> element.

Within the most common attributes accepted by this element there are: action and method. The action contain the URL (server) to which the form is going to be sent. Whilst, the method attribute is the HTTP (transfer protocol) the browser should use when submitting the form. Both of these <form> attributes pertain to submitting and processing the data.

Text fields

These elements allow gather data in the form of text via the <input> element, and uses the type attribute to define the kind of information to be captured within the control. The most popular type attribute value is text, which denotes a single line of text input. Also, along with the type attribute, is a good practice to include the name, in this way, is easier to identify the meaning of the data gathered by this control.

As self–contained element, the <input> element only uses one tag (opening) don't wrapping any other content, being its value given by its attributes. The initial attribute values for this element were: text and password (password input), but HTML5 brought recently new values for the type attribute, with the aim of expand the range of semantical meaning for inputs and enhance the controls for the users. Should a browser not understand one of these HTML5 type attribute values, it will automatically fall back to the text attribute value. The following is a list of the new HTML5 input types:

  • color
  • date
  • datetime
  • email
  • month
  • number
  • range
  • search
  • tel
  • time
  • url
  • week

Textarea

Text field work to capture character–based data, however, when string–based is required to be gathered, another type of element is necessary, namely the <textarea> element, it differs from the <input> element in that it can accept larger text passages spanning multiple lines; this element has start and end tags that wrap plain text. Because plain text is the only type of value that this element accepts the type attribute doesn't play here, but the name attribute is still used.

The <textarea> element has two sizing attributes: cols for width in terms of the average character width and rows for height in terms of the number of lines of visible text. However, the dimensions of this element is as expected, modified via the CSS width and height.

Multiple choice inputs & menus

Another way to collect data is through drop–down and multiple choice lists. Each of those with specific usages and purposes.

Radio buttons

This type of multiple choice list allow users to quickly choice one option form a small list. This type of lists are created from an <input> element which has been assigned a type attribute value of radio. Each radio button element should have the same name attribute value so all of the buttons within a group correspond to one another. As in the list with radio buttons, the user is selecting a predefined option, each <input> element will need to have a declared value. Additionally, to preselect a radio button for users it is possible to use the Boolean attribute checked.

  2. <input type=“radio” name=“day” value=“Saturday”> Saturday<br>
  3. <input type=“radio” name=“day” value=“Sunday”> Sunday

Drop–down lists

This type of lists is of great help when a long list of element to choose from is presented to the user. Employ another type of list, like radio buttons or check boxes isn't not only visually unappealing, but also unpractical specially on mobile devices.

To create a drop–list the <select> parent element and the <option> child element are used. As is expected that all options are all of the same nature, the name attribute is placed inside the <select> element instead. hence, the value attribute on each <option element corresponds to the name attribute on the <select> element. The text to be displayed to the user is then wrapped into an <option> opening and closing tag. Much like in the case of radio buttons, a preselected option can be marked up using the checked Boolean attribute.

  2.  <option value=“Friday” selected>Friday</option>
  3.  <option value=“Saturday”>Saturday</option>
  4.  <option value=“Sunday”>Sunday</option>

Hidden input

Hidden inputs provide a way to pass data to the server without displaying it to users. Hidden inputs are typically used for tracking codes, keys, or other information that is not pertinent to the user but is helpful when processing the form. This information is not displyed on the page; however, it can be found by viewing the source code of a page. It should therefore not be used for sensitive or secure information. To create a hidden input, the type=“hidden” attribute value is added in addition to the appropriate name and value attributes.

Form labels

In order to help users understand what is being asked of them, and/or how to properly fill out the form, the implementation of labels, fieldsets, and legends will help to guide them.

Label

This elements provide captions or headings for form controls, unambiguously tying them together, creating accessible forms for users and assistive technologies. Needless to say, labels should include text that describes the inputs or controls they pertain to. Labels could include a for attribute which ties the label to the id with the same value in the corresponding <input> element. This match allows users click on the <label> element to bring focus to the proper form control.

Example
HTML
  1. <label for=“username”>Username</label>
  2. <input type=“text” name=“username” id=“username”>
Result

The <label> elements can wrap form controls, such as radio buttons or check boxes. Doing so allows omission of the for and id attributes.

Example
HTML

  3. </label>

  6. </label>
  7. <label>
  8.  <input type=“radio” name=“day” value=“Sunday”> Sunday
  9. </label>
Result

Form attributes

There are different attributes that allow add properties, conditions, and behavior to the form, such as disabling controsl and adding form validation. The following attributes are amongst the most common.

Disable

This Boolean attribute turns off an element rendering useless for interaction or input. Elements with this attribute will not send any value to the server for form processing. When applied to a parent element the disabled attribute will affect the element's child elements. If the element has the type=“hidden” attribute value, the hidden Boolean attribute is ignored.

Example
HTML
  1. <label>
  2.  Username
  3. <input type="text" name="username" disabled>
  4. </label>
Result

If one tries to type any character in this disabled field it will be unresponsive.

Placeholder

This attribute shows information within the form control of an <input> or <textarea> that disappears once the element is clicked or gain focus. This is used to give users further information on how the form input should be filled in, for example, the email address format to use.

The main difference between the placeholder and value attributes is that the value attribute value text stays in place when a control has focus unless a user manually deletes it. This is a good option for pre-populating data, such as personal information, for a user but not for providing suggestions.

Example
HTML
  1. <label>
  2.  Email address
  3.  <input type="email" name="email-address" placeholder="name@domain.com">
  4. </label>
Result

Once any text is type in the above field the placeholder: name@domain.com will disappear.

Required

The required HTML5 attribute enforces that an element or form control must contain a value upon being submitted to the server, otherwise, an error message will be displayed by the browser, who handle the styling of this message, that is to say, this error message cannot be styled using CSS. Form controls and invalid elements on the other hand, can be styled using the :optional and :required CSS pseudo-classes.

Validation also occurs specific to a control's type. For example, an <input> control form with a type="date" will verify that the provided value is a valid one, and not, let's say text or symbols other than numbers.

Example
HTML
  1. <label>
  2.  Email Address
  3.  <input type="email" name="email-address" required>
  4. </label>
Result

Summary

Forms represent one basic way to get an interaction from the user, could modify a few element's set to serve multiple purposes, being the ones outlined here the following:

  1. The <form> element, the way to initialize form controls.
  2. The <input> element, being the most versatile so far, given that with the addition of a:
    1. <type="text" attribute a text field is introduced in the page.
    2. <type="radio" attribute will result in a radio button lists.
    3. <type="checkbox"> attribute will lead to a multiple selection list.
    4. <type="submit"> displays a form submission button.
    5. hidden attribute allow send relevant hidden information for the form that might be incomprehensible to the user.
  3. Accessibility can be enhance through the use of the <label> elements.
  4. The use of the <fieldset> element as a semantic sectioning element for form controls.
  5. How it is possible to change the state of a form control using the disable, required, and placeholder attributes.
