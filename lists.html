Lists

Intro

There are three types of HTML lists, namely, unordered lists, ordered lists, and description lists. As any HTML element its use depends mostly—if not exclusively—on its semantics or content meaning. Additionally, on the side of the CSS, lists can be styled in different ways. For example, the most common modified parameters in a list, include the indentation and the marker, this one can be squared, rounded, numeric, alphabetical, custom, or perhaps nonexistent. Also, besides the standard vertical list disposition, it is possible to arrange them horizontally.

Unordered lists

An unordered list, comprehends a collection of related items whose order does not matter. Creating an unordered list in HTML is accomplished using the unordered list block-level element, <ul>. Each item within an unordered list is individually marked up using the list item element: <li>.

The default's browser behaviour is to add some left padding and a vertical margin to the <ul> element and precede (again, in the case of unordered lists) each <li> element with a solid dot. This solid dot is called the list item marker, and it can be changed using CSS.

<ul>
 <li>Orange</li>
 <li>Green</li>
 <li>Blue</li>
</ul>

Ordered lists

This type of lists works pretty much the same as unordered list; individual list items are created in the same manner. The block-level element that denotes them is <ol>. Because order matters in here, instead of using a dot as the default list item marker, an ordered list uses numbers.

<ol>
 <li>Head north on N Halsted St</li>
 <li>Turn right on W Diversey Pkwy</li>
 <li>Turn left on N Orchard St</li>
</ol>
  1. Head north on N Halsted St
  2. Turn right on W Diversey Pkwy
  3. Turn left on N Orchard St

Ordered lists also have unique attibutes available to them including start and reversed.

Start attribute

This attribute defines the number from which a list should begin, as expected, its default value is 1. However, there may be cases where a list should start at a different number, in this case, the attribute is added to the <ol> tag in the form of an integer number, regardless of the numbering system selected, such as roman numerals.

<ol start="30">
 <li>Head north on N Halsted St</li>
 <li>Turn right on W Diversey Pkwy</li>
 <li>Turn left on N Orchard St</li>
</ol>
  1. Head north on N Halsted St
  2. Turn right on W Diversey Pkwy
  3. Turn left on N Orchard St

Reversed attribute

This attribute is used whenever a list in reverse order is desired—as long as we are working with an ordered list—this, is a Boolean attribute, and as such it doesn't accept any value. It is either true or false, being the latter the default value, whenever appear in the <ol> tag, this value becomes true.

<ol reversed>
 <li>Head north on N Halsted St</li>
 <li>Turn right on W Diversey Pkwy</li>
 <li>Turn left on N Orchard St</li>
</ol>
  1. Head north on N Halsted St
  2. Turn right on W Diversey Pkwy
  3. Turn left on N Orchard St

Notice from the previous example that the reversed attribute will reverse the numbering order, not the content order.

Value attribute

This attribute will make the numbering of a <li> element jump to the number or value selected in this attribute, any subsequent <li> will be numbered upwards from it.

<ol>
 <li>Head north on N Halsted St</li>
 <li value="9">Turn right on W Diversey Pkwy</li>
 <li>Turn left on N Orchard St</li>
</ol>
  1. Head north on N Halsted St
  2. Turn right on W Diversey Pkwy
  3. Turn left on N Orchard St

Description lists

Another type of lists no so commonly used is the description list, this type of lists are rather useful for creating glossaries and in a more general way, outline multiple terms and their descriptions. When doing so, the block-level element <dl> is used, and instead of using a <li> element to mark up list items, the description list requires of two block-level elements: the description term element, <dt>, and the description element, <dd>.

Besides the possibility to contain several description terms, <dt>, a description list may have several terms per description, i.e. a <dt> with multiple <dd>. Conversely, a single term may have several descriptions, that is, a <dd> have or be associated with multiple <dt> (think of a thesaurus). The order of these items, <dt> and <dd> is specially important in description list, here, the <dt> must come first than the <dd>.

<dl>
 <dt>study</dt>
  <dd>The devotion of time and attention to acquiring knowledge on an academic subject, especially by means of books.</dd>
 <dt>design</dt>
  <dd>A plan or drawing produced to show the look and function or workings of a building, garment, or other object before it is built or made </dd>
  <dd>Purpose, planning, or intention that exists or is thought to exist behind an action, fact, or material objects.</dd>
 <dt>business</dt>
 <dt>work</dt>
  <dd>A person's regular occupation, profession, or trade</dd>
</dl>

Nested lists

A powerful and useful feature of HTML lists is their capacity of being nested. Nevertheless, there are a few considerations to be taken into account when working with nested lists:

  1. The ability to indefinitely nest lists doesn't provide free rein to do so, they should be nested as long as they preserve their semantic value.
  2. It is only possible to nest a list—ordered or unordered—inside a <li> tag. Or in other words, the only child element of any <ol> or <ul> will be a list item, <li>.
  3. Any list item, <li>, that contains a nested list, must entirely hold the proper opening and closing tag of the nested list, prior to close the parent list item with the </li> tag. That is to say, once the nested list is complete and closed, the parent list item can be wrapped and continue with the original list.
<ol>
 <li>Walk the dog</li>
 <li>Fold laundry</li>
 <li>
 Go to the grocery and buy:
 <ul>
  <li>Milk</li>
  <li>Bread</li>
  <li>Cheese</li>
 </ul>
 </li>
 <li>Mow the lawn</li>
 <li>Make dinner</li>
</ol>
  1. Walk the dog
  2. Fold laundry
  3. Go to the grocery and buy:
    • Milk
    • Bread
    • Cheese
  4. Mow the lawn
  5. Make dinner

List item styling

List style type property

The list–style–type property is used to modify an item marker. The available values range from square and decimal numbers all the way to Armenian numbering, and the style may be placed on either the <ul>, <ol>, or <li> elements. With this in mind, it is possible to mix numeric markers into unordered list and nonnumeric markers on an ordered list. Take a look which are the other types of markers detailed in the link.

HTML

<ul>
 <li>Orange</li>
 <li>Green</li>
 <li>Blue</li>
</ul>

CSS

ul {
 list–style–type: square;
 margin–left: 40px;
}

Using an image as a list item marker

There may come a time when the default list–style–type property values are not enough. In such occasions, prior to add a custom one, is necessary to remove the set list marker for the <li> using the list–style–type: none; and add a background image, as usual, the rules to add background images will apply. Also, it might be necessary to add/include a padding.

HTML

<ul>
 <li>Orange</li>  <li>Green</li>  <li>Blue</li> </ul>
li {
 background: url("arrow.png") 0 50% no–repeat;
 list–style–type: none;
 padding–left: 12px;
}
