Introback to top

Text has always been a fundamental part of HTML, as time has passed, additional type of content like images, audio, video, and even other web page snippets, have been introduced to convey content in a more powerful way. Here is going to be explained how to add and use this media elements in the construction of a web site.

Imagesback to top

Adding images can be done in a pretty straightforward way by means of the <img> inline element. The <img> element is a self–containing, or empty, element; meaning that it doesn't wrap any other content, therefore it doesn't have a closing tag, in other words, there's no such a thing as </img> tag. For this element display the intended image is necessary to specify the path to it, which can be done using the src attribute, this is basically an url, typically relative to the server where the image or website is hosted. On the other hand, should the image be unavailable for any reason, an alt attribute (alternative text) describing and/or including keywords of the image must be added to the tag; this attribute is used for search engines and assistive technologies to help convey the purpose of it.

Example
HTML
  1. <img src="https://images.com/wolf.jpg" alt="A light grey wolf in the wilderness sniffing a photographic camera.">
Result

Supported image formatsback to top

Browsers may or may not support different types of images formats. By large, the most commonly supported formats online are jpg, png and gif formats. The jpg format is more suited for situations where the quality of a high color count image needs to be preserved whilst keeping a decent file size, such in a photograph on a website. On the other hand, png format is better for images with transparencies or low color counts, such in an icon or a background pattern.

Sizing imagesback to top

It is common to encounter situations in which is necessary to adjust the size of an image, this could be to fit an element's size, adjust it to the nearby content, or some other reason. Not only adjusting the size of an image can be of aesthetic value, it also could translate in faster loading times, telling the browser how large the image should be even before the page loads. There a few different ways to size an image:

  1. Specifying a value for the CSS property height but not width (preserves aspect ratio).
  2. Specifying a value for the CSS property width but not height (preserves aspect ratio).
  3. Specifying a value for both, width and height CSS properties (might or might not destroy aspect ratio).
Example
  1. Specifying a value for the CSS property height but not width (preserves aspect ratio).
HTML
  1. <img src="https://images.com/wolf.jpg" alt="A light grey wolf in the wilderness sniffing a photographic camera.">
CSS
  1. img {
  2.  height: 400px;
  3. }
Result
  1. Specifying a value for the CSS property width but not height (preserves aspect ratio).
HTML
  1. <img src="https://images.com/wolf.jpg" alt="A light grey wolf in the wilderness sniffing a photographic camera.">
CSS
  1. img {
  2.  width: 400px;
  3. }
Result
  1. Specifying a value for both, width and height CSS properties (might or might not destroy aspect ratio).
HTML
  1. <img src="https://images.com/wolf.jpg" alt="A light grey wolf in the wilderness sniffing a photographic camera.">
CSS
  1. img {
  2.  height: 400px;
  3.  width: 400px;
  4. }
Result

Positioning imagesback to top

The following are available ways to position images:

  1. As inline-level elements, which is its default value.
  2. Floating them.
  3. Using the CSS display property.
  4. Modifying the padding, border, and margin box model properties.

1. Inline images positioningback to top

As by default the <img> element is an inline element, hence, if added and no CSS property is specified the image would be displayed within the same line as the content that surrounds it. Additionally, the height of the line in which an image appears will be changed to match the height of the image, which can create large vertical gaps within that line. However, leaving images untouched in their default positioning isn't too common. More often than not, images are displayed as block–level elements or are floated flush to one side.

2. Block images positioningback to top

This technique assigns the block: display; property to the image, making the image be displayed in its own line surrounded by text above and below it.

Example
HTML
  1. <blockquote>
  2.  <p>&#8220;And when, on the still cold nights, he pointed his nose at a star and howled long and wolflike, it was his ancestors, dead and dust, pointing nose at star and howling down through the centuries and through him.&#8221;<img src="https://images.com/wolf-howling" alt="A gray wolf howling in a sunny day with rocky mountains behind him."></p>
  4.  <p><cite>The Call of the Wild &mdash; Jack London</cite></p>
  5. </blockquote>
CSS
  1. img {
  2.  display: block;
  3.  height: 50%;
  4. }
Result

3. Flush images positioningback to top

The previous two techniques, inline, and block positioning will introduce the image at a point in the element. However, if is necessary to position the image either to the left or right, while at the same time allowing surrounding content adapt and wrap around the image as required, a float must be used, taking advantage of course, that this is the original purpose of this property.

Floating an image is the first step, if no margin, border, or padding is added, the text will align directly against it.

Example
HTML
  1. <blockquote>
  2.  <p>&#8220;And when, on the still cold nights, he pointed his nose at a star and howled long and wolflike, it was his ancestors, dead and dust, pointing nose at star and howling down through the centuries and through him.&#8221;<img src="https://images.com/wolf-howling" alt="A gray wolf howling in a sunny day with rocky mountains behind him."></p>
  4.  <p><cite>The Call of the Wild &mdash; Jack London</cite></p>
  5. </blockquote>
CSS
  1. img {
  2.  background: #eaeaed; /* White */
  3.  border: 1px solid #9799a7; /* Silver */
  4.  float: right;
  5.  margin: 8px 0 0 20px;
  6.  padding: 4px;
  7. }
Result

When to use <img> vs backgroundback to top

As anything in HTML the usage of an element comes down to its semantic value. In the case of the <img> element, this is suited when the contained image is relevant to the content of the document.

On the other hand, the background property is the preferred option when the image being used is part of the design or user interface of the page. As such, it is not directly relevant to the content of the page.
