Imagesback to top

Adding images can be done in a pretty straightforward way by means of the <img> inline element. The <img> element is a self–containing (or empty), element; meaning that it doesn't wrap any other content, therefore it doesn't have a closing tag, because of this, there's no such a thing as </img> tag. For this element display the intended image is necessary to specify the path to it, this can be done using the src attribute, which is basically an url, typically relative to the server where the image or website is hosted. On the other hand, should the image be unavailable for any reason, an alt attribute (alternative text) describing and/or including keywords of the image must be added to the tag; this attribute is used for search engines and assistive technologies to help convey the purpose of the image or content. Supported image formats

Browsers may or may not support different types of images formats. By large, the most commonly supported formats online are jpg, png and gif formats. The jpg format is more suited for situations where the quality of a high color count image needs to be preserved whilst keeping a decent file size, such in a photograph on a website. On the other hand, png format is better for images with transparencies or low color counts, such in an icon or a background pattern. Check that the icon selected for the lists document has a png format.

Sizing imagesback to top

It is common to encounter situations in which is necessary to adjust the size of an image, this could be to fit an element's size, adjust it to the nearby content, or some other reason. Not only adjusting the size of an image can be of aesthetic value, it also could translate in faster loading times, telling the browser how large the image should be even before the page loads. There a few different ways to size an image:

  1. Specifying a value for the CSS property height but not width (preserves aspect ratio).
  2. Specifying a value for the CSS property width but not height (preserves aspect ratio).
  3. Specifying a value for both, width and height CSS properties (might or might not destroy aspect ratio).

Positioning imagesback to top

The following are available ways to position images:

  1. As inline–level elements, which is its default value.
  2. Floating them.
  3. Using the CSS display property.
  4. Modifying the padding, border, and margin box model properties.

1. Inline images positioningback to top

As by default the <img> element is an inline element, hence, if added and no CSS property is specified the image would be displayed within the same line as the content that surrounds it. Additionally, the height of the line in which an image appears will be changed to match the height of the image, which can create large vertical gaps within that line. However, leaving images untouched in their default positioning isn't too common. More often than not, images are displayed as block–level elements or are floated flush to one side. When to use <img> vs background

As anything in HTML the usage of an element comes down to its semantic value. In the case of the <img> element, this is suited when the contained image is relevant to the content of the document.

On the other hand, the background property is the preferred option when the image being used is part of the design or user interface of the page. As such, it is not directly relevant to the content of the page.

Audioback to top

In addition to images, another type of media that can be easily added to a web document is audio via the HTML <audio> element, unlike the <img> element, though, audio requires both opening and closing tags, accepting the following attributes:

  1. <src>, specify the source URL to the audio to be played.
  2. controls a Boolean attribute which has a value of true when present in the element, should it be added, it will display the browser's default audio controls, i.e. play, pause, seek, and volume. Otherwise, the element won't be visible in the document.
  3. loop Boolean attribute that will cause an audio file to repeat continually from beginning to end.
  4. preload helps identify what, if any, information about the audio file should be loaded before the clip is played. It accepts three values:
    • none, no information about the audio file is going to be preloaded.
    • auto, will preload all information about an audio file.
    • metadata, will preload any available metadata information about and audio file, such as the clip's length, but not all the information; sits in between the none and auto values.

If the preload attribute isn't present, all information about the audio file will be loaded as if the auto value was specified. For this reason, when an audio file is not essential to a page, is good practice to add the preload attribute with either a none or metadata value to help conserve bandwidth and allow the page load faster.

Example
HTML
  1. <audio src="http://traffic.libsyn.com/impossibleradio/Gym_Jones.mp3" controls preload="metadata"></audio>

Audio fallbackback to top

When using audio—and overall media—is necessary to keep in mind that not all browsers support all audio formats, considering this case, more than one source must be provided for an audio file, i.e. an audio fallback. To add a fallback the src attribute is deleted, and instead the <source> element is nested into the <audio> element.

Adding the type attribute inside the <source> element, helps the browser quickly identify the available audio types, the most common are: mp3, wav, and ogg. When a browser recognizes an audio file format it will load that file and ignore all the others.

Because it was introduced in HTML5, some browsers may not support the <audio> element. For those cases, a download link might be provided after any <source> elements within the <audio> element.

Videoback to top

To add a video to a web page the <video> element is used, their attributes, namely: src, controls, autoplay, loop, preload, and <source>. behaves exactly the same as the <audio> element, with the only exception that it counts with an additional attribute: poster and, that if the Boolean attribute controls isn't specified the video is going to still be displayed. Additionally, unless the autoplay Boolean attribute is applied, the visualization of the video might be restricted. If there is a good reason to not allow users to start, stop, or replay the video the controls attribute should be omitted.

As an additional consideration, given that videos will take up space, is highly recommended to specify their height and width via CSS. This helps ensure that video isn't larger than necessary, rendering it faster and within the document's layout. In regards of the fallback options, that could be used in place of a plain text fallback is to use a YouTube or Vimeo embedded video. These video hosting websites allow to upload videos, providing a standard video player, and enable to use embed videos onto a pages using an inline frame.

Inline framesback to top

HTML allows embed the content of another, external page into the current document. This is done using and inline frame: <iframe> element. It accepts the URL of another HTML page within the src attribute value; displaying the content from the embedded HTML page. The src attribute value may be a URL relative to the page the <iframe> element appears on, or an absolute URL for an entirely external page.

The default values for this element include an inset border, width and height, which can be modified using HTML with the frameborder, width, height attributes; or via CSS with the border, width, and height properties. Is necessary to take into account that as external resources:

  1. Pages referenced within the src attribute of an <iframe> element play their own rules, as they do not inherit any style or behaviors from the page they are referenced on.
  2. Other way around behavior it is also true, that is, the pages referenced on the <iframe> won't add any of the styles to the current document.
  3. Additionally, links within the page referenced inside the <iframe> element will open inside that frame, leaving the page that contains the <iframe> element unchanged.
  4. In other words, an <iframe> can be thought as a window or mini–browser to visualize other web pages.

Semantically identifying figures & captionsback to top

When there is present more than one item of self–contained content, specially media, such as multiple images, videos, diagrams, code blocks, audio clips, illustrations, diagrams, among others, all of those can be wrapped into a <figure> element; that were created to semantically mark up self–contained content or media, commonly with a caption (<figcaption>). Before HTML5 this was frequently done using an ordered list. While an ordered list worked, the markup was not semantically correct.

Figureback to top

This element is a block–level element, as mentioned above, is used to identify and wrap self–contained content, often in the form of media. It may surround:

  1. Images
  2. Audio clips
  3. Videos
  4. Blocks of code
  5. Diagrams
  6. Illustrations
  7. Other self–contained media.

More than one (media) item of self–contained content, such as multiple images or videos, may be contained within the <figure> element at a time, in such a way that if this element is moved from the main portion of a page to another location, it should not disrupt the content or legibility of the page.

Figure captionback to top

Semantically can be very useful to add a legend or description of the content grouped in the <figure> element, to do this the <figcaption> element should be used, although not all <figure> element must have a <figcaption> element, if its addition leads to a relevant description of all the items grouped into the <figure> element, then is appropriate to use it. As it describes all the grouped items, this element should be used only once, and could be located anywhere within the <figure> element (top, bottom, middle, etc.)

Additionally, the <figcaption> element may replace an <img> element's alt attribute if the content of the <figcaption> element provides a useful description of the visual content of the image. Include a relevant example of the use of the figure and figcaption below.

Summaryback to top

  1. Adding images and their description by means of the <img> element and alt attribute respectively.
  2. Inset audio specifying its source with either a relative or absolute URL, or providing the specific file in order to guarantee browser compatibility.
