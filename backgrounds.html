Background color

The HTML provides several options to custom the background of a page, either with color or with images. To do this, it is possible to use either the background or the bakcground-color properties. the former accepts color and images, but the latter is strictly used for solid colors, which can be specified using hexadecimals, RGB, RGBa, HSL, or HSLa values. The following is an example of a background-color being set:

div {
background-color: #b2b2b2;
}

Transparent colors & backwards compatibility

Olders browsers not always support the use of transparent colors, for those cases, is useful to take advantage of the way CSS works, applying in a first-arrived, first-served basis, based on this, and thinking for those browsers, is useful to first declare a background-color using hexadecimal values, and later using a transparent background-color within the same rule set, the first hexadecimal value will provide a "save" background color, whilst, the second will provide a transparent background if supported. Like this:

div {
background-color: #b2b2b2;
background-color: rgba(0, 0, 0, .3);
}

Background image

In addition to adding a background-color it is also possible to add an image as a background. For this purpose, the background or the background-image properties can be used outright. Regardless of the property used an image source must be specified using the familiar rules for hyperlinks, although this time instead of the href="" attribute an url("") attribute is used, which employs a quotation between parenthesis, note also that there's neither equal (=) nor colon (:) symbols preceding the parenthesis or the quotations. Is always necessary to keep an eye on the image source path/directories, to reference the desired imageAn example of a background-image property is shown below:

div {
background-image: url("alert.png");
}

Solely sourcing the image using an url value usually isn't enough, by default, the image will repeat itself from the top left corner of its parent element filling up the screen. To control this behavior the background-repeat and background-position properties can be used for this task.

Background-repeat

This property controls if an image is repeated as a background, and if it does, how it is done, for example, the direction of the repeat can be changed from its default left-to-right, or if it will repeated horizontally or vertically.

This property accepts four different values: repeat, repeat-x, repeat-y, and no-repeat, being the default repeat. The repeat-x value, will repeat the image horizontally only; the repeat-y will do it in a vertical fashion; lastly, no-repeat tells the browser to display the image just once.

Background position

By default, images are placed at the top left corner of an element, by using the background-position property it is possible to control exactly where the background image is placed relative to that corner.

div {
background-image: url("alert.png");
background-position: 20px 10px;
background-repeat: no-repeat;
}

In the above example, the first value of this property, corresponds to the horizontal offset, and the second one to the vertical offset. If at the time this property is declared, if only one value is stated it will be assumed as the horizontal offset; the vertical offset will default 50%. As the image is moved related to the top-left corner of its parent element, length values specifically will be in relation to that corner.

The background-position property accepts keywords, percentages, pixels, or any length measurement. Keywords (top, right, bottom, left, and center) and percentages are strongly related, for example, a keyword value of left top is identical to the 0 0 percentage; similarly, they keyword value right bottom is exactly the same as 100% 100%. Using pixels for position a background is also common, as they provide more precise control over where the background will be positioned.

Shorthand background image values

All background properties: background-color, background-image, background-position, and background-repeat, can be summarized in a single shorthand background property, like this:

div {
background: color image position repeat;
} div {
background: color image-properties;
} div {
background: #b2b2b2 url("alert.png") 20px 10px no-repeat;
}

The following example shows the set up a background image using shorthand values. Notice that for the position property there are both absolute and relative values. The first value, 20px, is the horizontal value, positioning the background-image 20px from the left of the element. The second value, 50%, is the vertical value, which vertically centers the background-image. A few other properties and values are also included within the notice-success class rule set to further style the alert message.

HTML

<div class="notice-success">
Woo hoo! Congratulations, you did it!
</div>

CSS

.notice-success {
background: #67b11c url("tick.png") 20px 50% no-repeat;
border: 2px solid #467813;
border-radius: 5px;
color: #fff;
font-family: "Helvetica Neue", Helvetica, Arial, sans-serif;
padding: 15px 20px 15px 50px;
}

Gradient backgrounds

HTML allows for the use of two colors and set up a linear or radial gradient. As expected the linear gradient, is controlled by the linear-gradient() function, as shown below:

div {
background: #466368;
background: -webkit-linear-gradient(#648880, #293f50);
background: -moz-linear-gradient(#648880, #293f50);
background: linear-gradient(#648880, #293f50);
}

A side note on vendor prefixes

The previous example shows the linear-gradient() function for the background property with prefixes for the different web-browsers, as the CSS3 introduced this option and have been progressively adopted, the need to specify these vendor prefixes has been reduced. However, in order to guarantee proper rendering is good option to include them. show demo

The previous code snippet shows that this function accepts two colors, the first one will be the top color and the second one will be bottom color. Browsers handle the transition between both, but additionally, to provide a fallback option should the browser don't support that gradient backgrounds a solid background color needs to be specified.

Changing the direction of a gradient background

By default background gradients go from top to bottom, nevertheless, the gradient orientation can be changed by the use of keywords, for example the keyword value to right will cause the gradient to move from the left to the right of an element. These keyword values can also be combined, orienting the gradient diagonally, as the to right bottom keyword does. show demo

div {
background: #466368;
background: linear-gradient(to right bottom, #648880, #293f50);
}

When a diagonal gradient is used in a non-square element, this will identify the element's absolute center and from there frame the element into a square/rectangle, applying the gradient from one corner to the other. These corners the gradient moves towards are called "magic corners". Further information can be gotten at Linear Gradient Keywordsshow examples

Besides keyword values, gradients can be also controlled by degree values from 0 through 360, for example, a degree value of 315deg will cause the gradient to move to the left top on an element; a 135deg will move the gradient to the right bottom of an element.

Radial gradient background

Just as linear gradients, radial gradients makes the transition between two different colors, being the first one the one in the center, and the second one the outer color. However, radial gradients, have additional parameters, with values for location, radius, size, and so forth. show examples

div {
background: #466368;
background: radial-gradient(#648880, #293f50);
}

The function will sit in the absolute center of the element, the browser will then create the transition between the two colors.
