Child selectors

This type of selectors allow target elements nested within a parent element. These selections can be made two different ways, using either descendant or direct child selectors.

Descendant selector

This is the most common selector, which matches every element that follows an identified ancestor. The selected element doesn't need to fall immediately after the parent element inside the document tree, but may fall anywhere within the ancestor element. Descendant selectors are created by spacing apart elements within a selector, creating a new level of hierarchy for each element list.

For example the section aside selector target any aside element nested inside the section element, regardless of the position of the former into the section element. Additionally, any aside element outside the section won't be selected.

In the following example the indicated aside elements are going to be selected.

HTML
  1. <aside>...</aside>
  2. <section>
  3.  <aside>This section will be selected</aside>
  4.  <figure>
  5.   <aside>This section will be selected</aside>
  6.  </figure>
  7. </section>
CSS
  1. section aside {...}

Direct child selector

To transform a child selector into a direct (child) selector a > is placed between the parent and the child selector. This will target only elements that fall directly within the parent element, that is, excluding any child element that is nested inside another type of element.

Continuing the previous example, the section > aside direct child selector will target the following elements.

  6.  </figure>

Sibling selectors

In addition to the possibility to target nested elements, there's also the option to target adjacent elements within the parent element, known as sibling selectors. These sibling selections can be made by way of the general sibling and adjacent sibling selectors.

General sibling selector

This selector allows target elements based on their sibling elements. Siblings elements are those who share the same parent element and are in the same hierarchy level in the document tree, they are created by using the tilde character: ~ between two elements within a selector. The first element identifies what the second element shall be a sibling with, and both of which must share the same parent element.

The h2 ~ p selector is a general sibling selector that looks for a p elements that follow, and share the same parent, of any h2 elements. In order for a p element to be selected it must come after any h2 element. Should a p element be nested into another element it won't be selected.

The paragraphs on lines 5 and 9 are selected as they come after the heading within the document tree and share the same parent as their sibling heading.

HTML
  1. <p>...<p>
  2. <section>
  3.  <p>...</p>
  4.  <h2>...</h2>
  5.  <p>This paragraph will be selected</p>
  6.  <div>
  7.   <p>...</p>
  8.  </div>
  9.  <p>This paragraph will be selected.</p>
  10. </section>
CSS
  1. h2 ~ p {...}

Adjacent sibling selector

There are situations in which is necessary select a sibling element that follows another sibling element, which is where the adjacent sibling element comes in. This selector will only target elements that directly follow (i.e. adjacent) the stated sibling element, instead of using the tilde ~ character between two elements (as in the case of general sibling selectors) the adjacent sibling selector uses a plus + character between the two elements within the selector. Just as before, the first stated element will specify the sibling element at the same tree level of the targeted (second) element.

From the example below can be observed that the selector h2 + p will select the p element that is immediately after any h2 element that is at the same level in the document tree:

HTML
  1. <h2>...</h2>
  2.  <p>...<p>

  4.  <p>...</p>
  5.  <h2>...</h2>
  6.  <p>This paragraph will be selected</p>
  7.  <div>
  8.   <p>...</p>
  9.  </div>
  10.  <p>...</p>
  11. </section>
CSS
  1. h2 + p {...}

This selector will look if an element have the stated attribute, but won't look at its value, just if the attribute is there. To select an element based on if an attribute is present or not, simply include a square brackets [] with the attribute name to look for in front of the element's name. The square brackets may or may not follow/include any qualifier such as an element type or class, all depending on the level of specificity desired.

Pseudo-classes

A pseudo-class, makes reference to a CSS selector that do not target an actual element, instead, is aimed to select dynamic elements as a result of user's actions. As seen in previous examples, one of the commonest pseudo-classes is the :hover pseudo-class, that targets elements whenever the pointer hovers over it. From this example, it also can be seen that this type of selectors begin with a colon ":", as will all other pseudo-classes.

List of pseudo-classes types
  1. Link
  2. Action
  3. State
  4. Structural
  5. Target
  6. Empty
  7. Negation

Link pseudo-classes

There are two basic pseudo-classes that comprehend this classification, namely: :link and :visited

:link
Style an anchor element that has not been visited.
Example: a:link {your rule set...}
:visited
Used to style anchor element that has been already visited by the user based on their browsing history.
Example: a:visited {your rule set...}
CSS
  1. a:link{...}
  2. a:visited {...}

User action pseudo-classes

This type of classes come into play whenever a user has specific types of interaction with different elements. They are:

:hover
Pseudo-class used to style elements whenever a user hovers over the specified element.
Example: table:hover {your rule set...}
:active
This pseudo-class will come into play whenever the user engages an element, such as clicking it.
Example: input:active {your rule set...}
:focus
Its styles will be used when a user makes an element the focus point of the page, often by using the keyboard to tab from one element to another.
Example: img:focus {your rule set...}
CSS
  1. table:hover {...}
  2. input:active {...}
  3. img:focus {...}

User interface pseudo-class

This type of classes targets elements depending on the state of some user interface elements, particularly within form elements. These user interface elements include:

:enabled
Styles an element when it is in its default state of enabled and available for use.
Example: input:enable. It might be useful to imagine the difference between an active element and when it is "gray out".
:disabled
Style an element or input when the disabled attribute is tied to it.
Example: input:disable {your rule set...} Most browsers will fade out disabled inputs to inform users that the input is not available for interaction, however, there's still room for customization of these styles with this pseudo-class.
:checked
This pseudo-class selector is specific to checkboxes and radio buttons input elements, selecting them if they are checked.
Example: input:checked {your rule set...}.
:indeterminate
Similarly to the :checked pseudo-class selector, this one it is also aimed to checkboxes and radio button. For this particular case, the styles within this rule set will be applicable only if these has neither been selected or unselected, living in an indeterminate state.
Example: input:indeterminate {your rule set...}
CSS
  1. input:checked {your rule set...}
  2. input:indeterminate {your rule set...}
