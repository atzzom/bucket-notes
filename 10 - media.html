Forms intro

Forms are used to retrieve information given by the user, process requests among others. Additionally, form gotten information can be used further by the user in a search query, a username or password. Here, the form main aspects are going to be explained, how do form elements are marked up, skipping the details about the back–end (server) interaction.

Initializing a form

To introduce a form the <form> element must be used, this identifies where on the page the form controls will appear, wrapping any child element within the form into an opening and closing tag, similarly to a <div> element.

Within the most common attributes accepted by this element there are: action and method. The action contain the URL (server) to which the form is going to be sent. Whilst, the method attribute is the HTTP (transfer protocol) the browser should use when submitting the form. Both of these <form> attributes pertain to submitting and processing the data.

Text fields

These elements allow gather data in the form of text via the <input> element, and uses the type attribute to define the kind of information to be captured within the control. The most popular type attribute value is text, which denotes a single line of text input. Also, along with the type attribute, is a good practice to include the name, in this way, is easier to identify the meaning of the data gathered by this control.

As self–contained element, the <input> element only uses one tag (opening) don't wrapping any other content, being its value given by its attributes. The initial attribute values for this element were: text and password (password input), but HTML5 brought recently new values for the type attribute, with the aim of expand the range of semantical meaning for inputs and enhance the controls for the users. Should a browser not understand one of these HTML5 type attribute values, it will automatically fall back to the text attribute value. The following is a list of the new HTML5 input types:

  • color
  • date
  • datetime
  • email
  • month
  • number
  • range
  • search
  • tel
  • time
  • url
  • week

Textarea

Text field work to capture character–based data, however, when string–based is required to be gathered, another type of element is necessary, namely the <textarea> element, it differs from the <input> element in that it can accept larger text passages spanning multiple lines; this element has start and end tags that wrap plain text. Because plain text is the only type of value that this element accepts the type attribute doesn't play here, but the name attribute is still used.

The <textarea> element has two sizing attributes: cols for width in terms of the average character width and rows for height in terms of the number of lines of visible text. However, the dimensions of this element is as expected, modified via the CSS width and height.

Multiple choice inputs & menus

Another way to collect data is through drop–down and multiple choice lists. Each of those with specific usages and purposes.

Radio buttons

This type of multiple choice list allow users to quickly choice one option form a small list. This type of lists are created from an <input> element which has been assigned a type attribute value of radio. Each radio button element should have the same name attribute value so all of the buttons within a group correspond to one another. As in the list with radio buttons, the user is selecting a predefined option, each <input> element will need to have a declared value. Additionally, to preselect a radio button for users it is possible to use the Boolean attribute checked.

Example
HTML
  1. <input type=“radio” name=“day” value=“Friday” checked> Friday<br>
  2. <input type=“radio” name=“day” value=“Saturday”> Saturday<br>
  3. <input type=“radio” name=“day” value=“Sunday”> Sunday
