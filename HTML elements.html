List of HTML elements

The following is the list of the HTML elements according to the World Wide Web Consortium (W3C).

HTML elements by function

1. Root element

<html> Root element. Represents the root of the document, all possible elements must be descendants of this element, that is to say that is the parent element of any possible element within the document.

2. Document metadata

Metadata provides information about the page like styles, scripts and data that help software as browsers, search engines, etc. to use and render the page.

<base> This will serve as the base for all relative URL contained within the document. There only can be one <base> element in the document.
<head> Document metadata container. Collects the document's metadata, providing the general information about the document, including its title and links to its scripts and style sheets.
<link> Represents metadata that expresses inter-document relationships (external resources). Possible uses for this element include defining a relational framework for navigation (what does this exactly mean/examples), and is mostly used to link to style sheets.insert examples of its usage.
<meta> A multipurpose element for representing metadata that cannot be represented by one of the other HTML meta-related elements (<base>, <link>, <script>, <style>, or <title>). Can have the following attributes: insert examples, turn the bullets into a list.

:
- name
- http-equiv=refresh
- http-equiv=default-style
- charset
- http-equiv=content-type

<style>: allows information to be embedded in documents. This information might affect the whole or part of the document. By default the style instructions written inside this element are expected to be CSS.insert examples of its usage.

<title>: define the title of the document shown in a browser's title bar or on the page's tab. It can only contain text, and andy contained tags are ignored.

3. Scripting

As pages added interactivity, web applications, and dynamic content, HTML supports the use of scripting languages, most prominently JavaScript. The follwing elements support this type of features:

<canvas>: element used to draw graphics via scripting (usually JavaScript), make photo compositions or even perform animations. Is strongly suggested to provide alternate content inside the <canvas> block so the information will be displayed in older browsers that don't suppport this feature or in browsers with JavaScript disabled.

<noscript>: defines a section of HTML elements to be inserted if a script type is unsupported or if scripting is currently turned off in the browser.

<script>: used to embed or reference an executable script within an HTML or XHTML document.

4. Sections

The content sectioning elements allow organize the document content into logical pieces, creating an outline for the content, including header and footer navigation, and heading elements to identify sections of content.

<address>: this element provides contact information for its nearest <article> or <body> ancestor; in the latter case, it applies to the whole document. see examples.

<article>: represents a self-contained content or composition in a document (page, application, or site), which is intended to be independently distributable or reusable (e.g. in syndication). Such as, a forum, blog post, newspaper or magazine article, an object, or any other independent item of content. Each <article> should be identified, typically by including a heading (<h1>—<h6> element) as a child of the <article> element.

<aside>: this element is intendend to hold information tangentially connected to the rest of the document or section, but that can be considered separate from that content. For example, a glossary, information/biography of the author, web applications, profile information or related links.

<footer>: this element represents the footer for its nearest sectioning content or nearest sectioning root element. It typically contains information about the author of the section, copyright data or links to related docuemnts.

<header>: represents a group of introductory or navigational aids. It may contain, a logo, wrapped section's header, search bar/form, and so on.

<h1>, <h2>, <h3>, <h4>, <h5>, <h6>: heading elements implement six levels, being <h1> the one with the greatest importance and <h6> the one with the least. Is specially useful to identify content sections and build automatically generated table of contents.

<hgroup>: (Heading Group) represents the heading of a section. It defines a single title that participates in the outline of the document as the heading of the implicit or explicit section that it belongs to. insert examples.

<nav>: (navigation) this element links to other pages or parts within the page, a section with navigational links.

<section>: contains a thematic grouping of content, typically with a heading. Each <section> should be identified, typically by including a heading as a child element.

Text content

The text content elements help organizing sections, blocks and pieces of information within the opening <body> and closing </body> tags. Also, are important for accessibility and SEO, these elements identify the purpose or structure of that content

<dd>: (description) indicates the description of a term in a description list (<dl>). insert examples.

<div>: (division) this element does not inherently represent anything (i.e. semantically meaningless), is a genertic container, and is used to group elements for styling purposes (using class and/or id attributes), or because they share attribute values, such as lang. It should be used only when no other semantic element —such as <article> or <nav>— is appropriate.

<dl>: description list common uses for this element are: glossaries, display metadata (a list of key-value pairs). Basically encloses a list of groups of terms and descriptions.

<dt>: description term identifies a description term inside a description list, this element can only appear as a child element of a <dl>. It it usually followed by a <dd> element; however, multiple <dt> elements in a row indicate several terms that are all defined by the immediate next <dd> element.

<figcaption>: as its name implies it, represents a caption or a legend associated with the image of the <figure> element that is its immediate ancestor, meaning that <figcaption> can be the first or last element inside a <figure> block. Also, the HTML figcaption element is optional; if not provided, then the parent figure will have no caption.

<figure>: this elements commonly holds content such as images, illustrations, diagrams, code snippets, or schemata referenced in the main text, but that can be moved to another page or to an appendix without affecting the main flow. In a more general way, the <figure> element represents self-contained content, frequently with a <figcaption> element, and typically referenced as a single unit. Its position is independent of the main flow.

<hr>: thematic break represents a thematic break between paragraph-level elements (for example, a change in a scene in a story, or a shift of topic with a section). Previously, in older HTML versions, was represented as a horizontal rule, and it may still be displayed as such in visual browsers which other types of browsers there are?, but is now defined in sematic terms (thematic break), rather than presentation terms.

<li>: list item used to represent an item inside a list, because of its nature/definition, must be contained into a parent element: an ordered list (<ol>) or inside an unordered list such as: <ul> or a menu: <menu>. In the latter, list items are usually displayed using bullet points, whilst in ordered list, they are usually displayed with an ascending counter on the left (number or letter).

<main>: contains the core/main content of the document, or expands upon the central topic of a document or central functionality of an application, being unique to the document, as such, must exclude repeating content across a set of documents such as sidebars, navigation links, copyrigh information, site logos, and search forms (unless the document's main function is a search form).

<ol>: ordered list holds an ordered list of items. Typically the ascending counter can be numbers, Romans numerals, letters, or simple bullets. This numbered style is defined in the associated CSS, using the list-style-type property.

<p>: paragraph this element represents a paragraph of text.

<pre>: preformatted text. Text within this element is displayed in a non-proportional ("monospace") font exactly as it is laid out in the file. Whitespace inside this element is displayed as typed. show usage examples.

<ul>: unordered list represents an unordered list of items, or where their order is meaningless/pointless. These items are usually represented with a bullet, which can be of several forms: dot, circle, square, etcetera. The bullet style is defined with the CSS property list-style-type.

Inline text semantics

The following elements are used to define the meaning, structure, or style of a word, line, or any arbitray piece of text.

<a>: anchor creates a hyperlink to other web pages, files, or locations within the same page, e-mail addresses, or any other URL.

<abbr>: abbreviation represents an abbreviation and optionlly provides a description for it. If present, the title attribute must contain this full description an nothing else. show examples.

<b>: bold this element is used for text that is stylistically different from normal text, without conveying any special meaning or importance. It is usually used for keywords in a summary, product names in a review, or other spans of text whose typical representation would be boldfaced. Another use, is to mark the lead sentence of each paragraph of an article.

<bdi>: bi-directional isolation isolates a span of text that might be formatted in a different direction from other text outside it. show examples.

<bdo>: bi-directional override used to override the current directionality of text. It causes the directionality of the characters to be ignored in favor of the specified directionality.

<br>: break (line) produces a break in the text (carriage-return). Useful for writing poems, addresses, where the division of lines is significant.

<cite>: cite represents a reference to a creative work, it must include the title of a work or a URL reference, which may be in an abbreviated form according to the conventions used for the addition of citation metadata. include examples.

<code>: represents a fragment of computer code, and is represented in the browser's default monospace font.

<data>: this element links a given content with a machine-readable translation. If the content is time- or date-related, the <time> element must be used.

<dfn>: definition represents the defining instance of a term.

<em>: emphasis marks text that has stress emphasis. The <em> element can be nested, with each level of nesting indicating a greater degree of emphasis. show examples of this.

<i>: italic represents a text segment that is set off from the normal text for some reason, for instance, technical terms, foreign language phrases, or fictional character thoughts. It is displayed in italic type.

<kbd>: keyboard represents user input and produces an inline element displayed in the browser's default monospace font.

<mark>: represents highlighted text, i.e., a run of text marked for reference purpose, due to its relevance> in a particular context. For example can be used in a page showing search results to highlight every instance of the searched-for word.

<q>: quote indicates that the enclosed text is a short inline quotation. This element is intended for short quotations that don't require paragraph breaks; for long quotations use the <blockquote> element.

<rp>: ruby parenthesis? this element is used to provide fall-back parenthesis for browsers non-supporting ruby annotations. Ruby annotations are for showing pronunciation of East Asian characters, like using Japanese furigana or Taiwainese bopomofo characters. The <rp> element is used in the case of lack of <ruby> element support its support its content has what should be displayed in order to indicate the presence of a ruby annotation, usually parentheses. show examples.

<rt>: ruby pronunciation? embraces pronunciation of characters presented in a ruby annotations, which are used to describe the pronunctiation of East Asian characters. This element is always used inside a <ruby> element. show examples.

<rtc>: embraces semantic annotations of characters presented in a ruby of <rb> elements used inside of <ruby> element. <rb> elements can have both pronunciation (<rt>) and semantic (<rtc>) annotations. show examples.

<ruby>: this element represents a ruby annotation. Ruby annotations are for showing pronunciation of East Asian characters. show usage examples.

<s>: strikethrough renders text with a strikethrough, or a line through it. Its usage must be selected to indicate when a portion of text is either unaccurate or irrelevant, and must be avoided to indicate text or document edition; for that, use the <del> and <ins> elements, as appropriate.

<sample>: sample, used to identify a computer's program sample output. It is usually displayed in the browser's default monotype font (such as Lucida Console).

<small>: makes the text font size one size smaller (for example, from lager to medium, or from small to extra-small) down to the browser's minimum font size. In HTML5, this element is repurposed to represent side-comments and small print, including copyright and legal text, independent of its styled presentation.

<span>: this element doesn't hold a semantic meaning, is a generic inline container for phrasing content, which does not inherently represent anything. It can be used to group elements for styling purposes (using the class or id attributes), or because they share attribute values, such as lang.

<strong>: as opposed to the <b> (bold) element, this element has a semantic meaning, gives text strong imporntance, and is typically dsiplayed in bold.

<sub>: subscript, defines a text span that should be displayed, for typographic reasons, lower, and most of the times, smaller than the main span of text.

<sup>: superscript, defines a text span that should be displayed, for typographic reasons, higher, and most of the times, smaller than the main span of text.

<time>: represents either a 24-hour clock time, or a precise date in the Gregorian calendar (with optional time an timezone information).

<u>: underline renders text with an underline.

<var>: variable, represents a variable in a mathematical or programming context.

<wbr>: word break opportunity represents a position within the text where the browser may optionally break a line, though its line-breaking rules would not otherwise create a break at that location.

Image and multimedia

The following elements are used to support various multimedia resources, such as, images, audio, and video.

<area>: this element can only be used within a <map> element, and represents a hot-spot region on an image, optionally, a hyperlink can be associated with it.

<audio>: allows embed sound content in documents. It may contain more than one source (represented by the src or source elements), the browser decides the most suitable one.

<img>: image, represents an image in the document.

<map>: is used with <area> elements to define an image map, i.e. a clickable link area.

<track>: used a child element of the <audio> and <video> media elements. It lets you specify timed text track (subtitles) or time-based data. The tracks are formatted in WebVTT format (.vtt files) - Web Video Text Tracks.

<video>: embeds video content in a document. As the <audio> media element, can contain more than one source, specified by the src attribute or the <source> element, again, the browser choose the most suitable one.

Embedded content

In addittion to the media content, HTML allows for other elements that support or complement it, even if it's not always easy to interact with.

<embed>: represents a plug-in, that is, an interaction point for an external application or interactive content.

<object>: represents an external resource, that can be: treated as an image, a nested browing context, or another resource to be handled by a <embed> (plug-in) element.

<param>: this element defines parameters for the <object> element.

<source>: specifies multiple media sources for either the: <picture>, <audio>, or the <video> element. It is an empty element, commonly used to serve the same media content in multiple formats supported by different browsers.

Scripting

With the purpose to add and support dynamic and interactive content in web applications, HTML supports the use of scripting languages, most prominently JavaScript. Certain elements support this capability.

<canvas>: usually used to draw graphics, photo compositions, even, perform animations, via scripting (e.g. with JavaScript). You may (and should) provide alternate content inside this block-level element, in order to support older browsers that might not render this conten and in browsers with JavaScript disabled.

<noscript>: used to insert an HTML section in the case a script is unsupported or if scripting is turned off in the browser.

Demarcating edits

The following elements provide indications that specific parts of the text have been altered.

<del>: represents a range of text that has been deleted from a document. Is often (but need not be) rendered as a strike-through text.

<ins>: inserted, represents a range of text that has been added to a document.

Table content

This elements are used and created to handle tabular data.

<caption>: this element serves as the title of the table, although is always the firts descendant of a <table>, its styling, using CSS, may place it elsewhere, relative to the table.

<col>: column defines a column within a table and is used for defining common semantics on all common cells. Generally found within a <colgroup> element.

<colgroup>: defines a group of columsn within a table. show examples of this.

<table>: used for information represented in a tabular way, that is, information expressed via a two-dimensional data table.

<tbody>: groups one or more <tr> elements at the body of a <table> element.

<td>: table data defines/set-up a cell that contains data. It participates in the table model. show examples of this and search what is the table model.

<tfoot>: defines a set of rows summarizing the columns of the table. show examples.

<th>: table header defines/identifies a cell as header of a group of table cells. The exact nature of this group is defined by the scope and headers attributes.

<thead>: defines a set of rows as the head of the columns of the table.

<tr>: defines a row of cells in a table. Those can be a mix of <td> and <th> elements. show examples.

Forms

HTML allows the use of different elements which can be used together to create forms which the user can fill out and submit to the web site or application. There's a great deal of further information about this available in the HTML forms guide.

<button>: represents a clickable button.

<datalist>: contains a set of <option> elements that represent the values available for other controls.

<fieldset>: used to group several controls as well as labels (<label>) within a web form. show examples.

<form>: represents a document section that contains interactive controls to submit information to a web server. show examples

<input>: create an interactive control that allow user input. Its behavior works depending on its type attribute.

<label>: represents a caption for an item in a user interface.

<legend>: represents a caption for the content of its parent element <fieldset>.

<meter>: represents either a scalar value within a known range or a vractional value. show example

<optgroup>: creates a grouping of options within a <select> element.

<option>: this element represents the result of a calculation or user action.

<progress>: this element is used to represent the completion progress of a task, typically displayed as a progress bar.

<select>: this element represents a control that provides a menu of options.

<textarea>: represents a multi-line plain-text editing control.

Interactive elements

These HTML elements are used to create interactive user interface objects.

<details>: this element is used as a disclosure widget from which the user can retrieve additional information. show examples.

<dialog>: represents a dialog box or toher interactive component, such as an inspector or window. show examples

<menu>: used to represent a group of commands that a user can perform or activate. This includes both list menues, which might appear across the top of a screen, as well as context menus, such as those that might appear underneath a button after it has been clicked. show examples

<menuitem>: represents a group of commands a user can invoke via a popup menu, this includes: context menus, as well as menus that might be attached to a menu button.

<summary>: for the content of the <details> element is used as a summary, caption, or legend. show examples
