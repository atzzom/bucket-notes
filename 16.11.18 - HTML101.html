What is HTML?

HTML (HyperText Markup Language), gives structure and meaning to the content of a document by defining that content as: paragraphs, headings, images, among others elements, the most important among them are: elements, tags, and attributes.

On the other hand, CSS (Cascading Style Sheets), is a language in charge of the presentation and style of the content, using colors, fonts etc.

Elements

Elements define stucture and content of objects within a page. Elements are identified by the use of less-than and greater-than angle brackets < >, surrounding the element name. Some of the most used elements are: multiple levels of headings (<h1>, <h2>, <h3>, ... , <h6>) paragraphs (<p>), links (<a> anchor), divisions (<div>) and many more (see list of HTML elements and the Mozilla Development Network).

Tags

The use of less-than and greater-than angle brackets surrounding and element creates what is known as a tag. Most of the times tags occur in pairs of opening and closing tags. An opening tag marks the beginning of an element (e.g.: div, p, a, h1, h2, h6, span, etc.), It consists of a less-than sign followed by the element's name with a greater-than sign, i.e. <element's name>, e.g. <h1>, <div>.

A closing tag marks the end of an element, and it consist of a less-than sign followed by a forward-salsh and the element's name with the greater-than sign, i.e. </element's name>. Therefore, a division tag will look like this: <div> ... </div>.

Attributes

An Attribute provides additional information about an element. The most common attributes are:

id: which identifies an element.
class: which classifies and element.
src: which specifies the source for embeddable content.
href: which provides a hyperlink reference to a linked source.

A characteristic of the attributes is that they are defined within the opening tag, and after and element's name.. Generally attributes includes a name and a value. The format of these attributes consists of the attribute name followed by and equal sign and then a quoted attribute value, for instance, an anchor element <a> with a href attribute would look like the following: <a href="google.com">just another search engine</a>

Understanding common CSS terms

In the same way there are terms commonly used in HTML, in CSS there are terms that appear repetively. These terms include: selectors, properties, and values.

Selectors

A selector allows target specific HTML elements within the document to style with properties such color, size, position, etc. They may include a combination of specific qualifiers in order to select unique elements, all depending on how specific we wish to be. For instance, we may want to select all the paragraphs of the document, or, only a single paragraph of the document. They generally target an attribute value, such as an id or class value, or target a type of element such as <h1> or <p>. CSS selectors use curly brackets, {}, which encompass the styles to be applied to the selected element. For example, the following selector is targeting all <p> elements:
p {...}

Properties

Properties are how elements are styled. Properties names fall after a selector, within the curly brackets {}, and immediately preceding a colon, :. Within the properties we find: color, background, font-size, height, among others. The following example shows two properties applied to all <p> elements of a document.

p {
color: ...;
font-size: ...;
}

Values

Represent how a property is set, for example, when applying the colorproperty to a text, is possible to set its value to blue. Values, fall after a colon : and a semi-colon ; just right after the property name, like this:
p {color: blue;}

Working with selectors

The most common ways to select the different elements in a HTML document is with the type, class, and ID selectors, that target the aforementioned elements based on the nature, classification and/or identity of them. Are there additional CSS selectors?

Type Selectors

They target elements based on their nature. For example, should we wish to target all paragraphs (<p>) in a document, we use then the type selector p -the whole CSS structure would be: p {property: value;}-.

Class Selectors

A classification selector, or class selector is a little more specific than type selectors, as they select a particular group of elements rather than all elements of one type, and allow us to select multiple elements of any kind containing the same class attibute. For example: the following class selector will target the following elements even though they are of different type:

HTML

<div class="awesome"> ... </div>
<p class="awesome"> ... </p>

CSS

ID selectors are even more specific than class selectors, just as them, ID selectors, use the HTML attribute ID (within the tag) as a selector. Regardless of the type of element they appear on, this selectors can be used once thorought the document why?, hence, if used, ideally they must be reserved for significant elements.

Getting to know HTML

Semantics overview

As we start building websites, is necessary to know which type of element is best to display an specific type of content, additionally, is very useful to know how elements are displayed on a web page.

Semantics is the meaning of a word from its name. Within HTML, is the practice to give the document the proper meaning and structure by using the adequate element. Semantic content works in the same way, describes its meaning and value regardless the style or appearance of that content. Semantic content has several benefits related with make easier for people, developers, computers, search engines, etc. to understand and manage the content on a web page as it shows clearly what piece of content is about. Before go deeper in this concepts, the elements <div> and <spans> -that contradictorily don't hold any semantic value, as they exist only for styling purposes- are going to be studied.

Identifying divisions & spans

This two elements do not have any specific meaning, that's why they don't hold any semantic value, they act as containers only for styling purposes. In order to see more clear the difference of the elements with semantic value from the ones who don't have any semantic value, let's see an example: Paragraphs (<p>) have semantic value as whatever the content they have (or are expected to have) is going to be interpreted as a paragraph (words), therefore, no other value is expected for this element (e.g. no images, video, media, etc. is expected); but on the other hand, an element as a <div> and a <spans> doesn't have this difference, basically, any type of content can be expected inside them (they are simply containers).

Although they are part of the HTML and not CSS, these elements are extremely valuable when building a website, as they give us the ability to apply targeted styles to a contained set of content. A <div> is a block-level element that encloses or identify large groupings of content, helping to build a page layout and design. On the other hand an <span> is an inline-level element used to identify smaller groups of text within a block level element.

Is common to see <div> and <spans> with class and id attributes with the purpose of aiding in the application of an style, however, is necessary to keep in mind that the value of these attributes ideally must refer to the content or nature of the element rather than its apperance or style.

For example, if we have a <div> with an orange background that contains social media links, our first thought might be to give the <div> a class value of orange. What happens if that orange background is later changed to blue? Having a class value of orange no longer makes sense. A more sensible choice for a class value would be social, as it pertains to the contents of the <div>, not the style.

The following is an example of the use of a division:

<p>I may be found on ... </p>
<p>Additionally, I have a profile on ...</p>
</div>

<!-- Span -->

Building structure

Webpages, used to be built using divisions to separate content, the problem was that these divisions didn't add any semantical meaning, and didn't give an idea about the content they holded. Because of this, structural based elements where introduced to help build web documents in an easier and more coherent way. All of these, in addition to give meaning, improve the semantics of the document. Examples of those elements are: <header>, <nav>, <article>, <section>, <aside>, and <footer> elements.

Header

As its name implies it, the <header> element, is used to identify the top of the document, whether it is the body, article, section, or other page segment. In general, the <header> element may include a heading, introductory text, and even navigation.

<header> vs. <head> vs. <h1> through <h6> elements.

<header>
Falls within the <body> element. Outlines the heading of a section/segment of the page.

<head>
Falls directly within the <html> element. Outlines metadata, the document title and links to external files.

Headings elements <h1> through <h6>
Falls within the body of the document, designate multiple levels of text.

Navigation

The <nav> element identifies a section of major navigational links on a page(might be thought as a section/division as a header, footer, aside, etc, etc.?). This element should be reserved for primary navigation sections only, such as global navigation, a table on contents, previous/next links, or other noteworthy groups of navigational links. Commonly the <nav> element links to other pages within the same website or parts of the same page. However, miscellaneous one-off links should be wrapped within the <a> (anchor elment) and the anchor element only.

Whenever there is self-contained or independent content that can be distributed or indepentently resused without confusion, such as, newspapers articles, blog post, user submitted content and the like, the <article>element should be used. A good criteria for its use can be, for instance, if the content within the <article> element were removed from the context of the page and placed, within an email or printed work, that content should still make sense.

The <section> element is commonly used to break up and provide hierarchy to a page.

Deciding between <article> , <section> , or <div> elements

At times is fairly difficult to decide which element is the best for the job, either the <article>, the <section> or the div element. As any matter related to every semantic decision, the key is to look at the content. In a nutshell:

If the content can be placed independently relocated without loose meaning, use the element <article>.

If the content represent a thematic group of content and adds to the document outline, then use the section element. Keep in mind that both, the <article> and <section> element contribute to a document's structure and help to outline a document.

But in the other hand, the displayed content is wrapped only for stylish purposes use the <div> element.

The <nav> element should be used for major navigational links on a page i.e. not small section details. such as global navigation, table of contents, previous/next links, or other noteworthy groups of navigational links. The <nav> element also identifies a section of major navigational links on a page. In other words, the <nav> elements behaves a a milestone within the website for the main navigational sections.

On the other hand, to handle miscellaneous one-off links the <a> (anchor) element should be used instead, and the anchor element alone.

Article

The <article> element is used to identify a section of independent, self-contained content that can be distributed or reused througout the website without lose meaning, like: blog post, user-submitted content, newspaper articles, and the like. The key to determine if it is appropiate to use the <article> element is to determine if that content could be extracted from the context of the page and placed, for example, within an e-mail or printed work, and should still make sense.

The <aside> element holds content that is tangentially related to the content surrounding it, such as sidebars, brief explanations, inserts, etc. For instance when used within an <article> element, for example, the <aside> element may identify content related to the author of the article. Instinctively the <aside> element might be thought to be placed in the right or left side of a page. However, as block-level element will fill up the whole page or parent element width unless stated otherwise.

Footer

This element identifies the closing or end of an: element, page, article, section, box, or other segement of a page. As its name suggest, is located at the bottom of its parent element. The content holded by the <footer> should be related information and should not diverge from the document or section it is included within.

Getting to know CSS

CSS is the language in charge of provide with format and styles to the structural HTML elements in the webpage, it stands for Cascade Style Sheet, and pours the different format from the top to the bottom of the document. Therefore, styles can be overwritten as the style sheet progresses.

For example, at the begining of the casade (or at the begining of the CSS document) we the the paragraphs elements to have a blue background color, but later in the CSS document, the color is set up to green, then, the paragraphs are going to be rendered green.

p {
background: blue;
font-size: 24px;
}
p {
background: green;
}

Because the paragraph selector that sets the background color to green comes after the paragraph selector that sets the background color to blue, it will take precedence in the cascade. All of the paragraphs will appear with a green background. the font size will remain 24 pixels because the second paragraph selector didn't identify a new font size.

The same effect will take place with the properties inside the individual selectors, as the cascade also pour them inside them, like this:

p {
background: blue;
background: green;
}

As mentioned previously, the styles of the cascade will be cascade from the top to the bottom of the style sheet, but not always will be this way. Those times occur when different types of selectors are used and the specificity of those selectors breaks the cascade.

Calculating Specificity

In CSS each selector has a precedence/specificity, this, along with its placement in the cascade, determines how its styles will be rendered. The type, class and ID selector, each has a different specificity weight.

The lowest specificity weight is hold by the type selector with a value of 0-0-1. The class selector has a medium specificity weight and holds a point value of 0-1-0. Lastly, the ID selector has a high specificity weight or precedence with a value of 1-0-0. Notice that the specificity/precedence is calculated using three columns. The first column counts ID selectors, the second columns counts class selectors, and the third column counts type selectors.

The specificity also can override a previously applied style, for example, if a paragraph is styled using a type selector in a first place, and later in the cascade is selected using a ID selector, as the latter has bigger precedence/specificity than the type selector, is going to override the type selector value regardless of its position in the cascade.

As the specificity values breaks the linearity of the cascade, although useful in many cases, it is also the cause of the styles not been rendered as intended.

Combining selectors

Individual selectors as type, class, and ID, can be combined together to be more granular and specific in the elements selection. For example, if within one element classified as hotdog we want to set all the paragraphs to have a brown background color. However, if one of those elements happens to have the class attribute of mustard, we want to set its background color to yellow. The HTML and CSS may look like the following:

<p<...</p<
<p<...</p<
<p class="mustard"<...</p<

background: brown;
}

background: yellow;
}

When selectors are combined they are read from right to left. The selector inmediately before the curly brackect, is known as the key selector, because identifies exactly which element the styles will be applied to, for example, in the following scenario:

HTML

<div class="hotdog">...</div>
<a class="mustard" href="mustard.com">mustard</a>
<p class="ketchup">...</p>
<p class="hotdog">...</p>
<footer class="end">...</footer>

CSS

.hotdog p {
background: brown;
}
.hotdog p.mustard {
background: yellow;
}

The first combined selector above, .hotdog p, includes two selectors: a class and a type selector, separated by a single space. The key selector targets <p> elements, but as is prequalified/preceded by the class selector .hotdog, this combined selector is going to chose: paragraphs element THAT are classified as/with hotdog.

The second selector .hotdog p.mustard is basically the same as the first one, with the addition of the class selector .mustard. This selector, is targeting paragraphs elements that are classified as mustard AND are nested into elements classified as hotdog. The absence of the space within these selectors would target: paragraphs elements classified as both: mustard AND hotdog at the same time, instead of nested. Following best practices, is desired not to prefix a class seletor (specificity 0-1-0) with a type selector (specificity 0-0-1), therefore if the best practice would be followed the selector should look like: .hotdog .mustard

Specificity with combined selectors

Work out the specificity in the case of combined selector is pretty straighforward, for example, in the combined selector: .hotdog p.mustard it has one type selector (0-0-1), and two class selectors (0-1-0), therefore its specificity is 0-2-1 (two class and one type selector). Therefore we can deduce that the specificity of .hotdog p.mustard will be exactly the same as the one for: p.mustard .hotdog, or .hotdog .mustard p, or p .hotdog.mustard, etc, even though these combined selectors might deliver different results. These permutation in the specificity calculation translates to the fact that we can change the order of these selectors (.hotdog p {} and .hotdog p.mustard) within our style sheet, placing the higher-weighted selector above the lower-weighted selector leading to the same result due to each selector's specificity weight. In general is desirable keep low specificity/precedence values as the higher the specificity the more likely the cascade is to break.

Layering styles with multiple classes

One way to keep the specificity as low as possible is to construct the website as modular as possible, to achive this, we can take advantage of the fact that HTML elements can have more than one class attribute as long as they are separated with a space, for example, the following element is classified as interactive, code, and hello-world:
<article class"interactive code hello-world">Some content here.</article>.
With these classifications is possible to place/layer certain style on all elements of one sort while placing/layering other styles only on specific elements of that sort.

Let's take a look at buttons, for example. Say we want all of our buttons to have afont size of 16px, but we want the background color of our buttons to vary depending on where the buttons are used. We can create a few classes and layer them on an element as necessary to apply the desired styles.

HTML

<a class="btn btn-danger">...</a>
<a class="btn btn-success">...</a>

CSS

.btn {
font-size: 16px;
}
.btn-danger {
background: red;
.btn-success {
background: green;
}
