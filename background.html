Backgrounds & gradients

Table of contents

Intro

The use of a background can help improve the overall website layout and usability, adding priority to content and helping to create a site's look and feel. CSS allows to set up different types of backgrounds, like solid colors, images, gradients or combinations of these.

Background color

The quickest way to add a background is by using a solid color, this can be done using the background or background–color property. The former, allows colors and images in shorthand form, while the latter, is strictly used for solid background colors.

div {
background–color: #b2b2b2;
}

<div> element with background–color: #b2b2b2; and height: 100px;

In the selection of the color, it is possible to use hexadecimal, RGB, HSL, RGBa, or HSLa values, allowing the use of transparencies.

Transparent backgrounds

Specially when background compatibility with older browsers is a concern such as IE8 that do not support transparent colors (RGBa, HSLa), it is useful to provide a fallback color. CSS cascades from the top to the bottom of a file; thus, it is possible to use two background–color properties within the same rule set. The first background–color property will use a "safe" background–color, such as a hexadecimal value, and the second background–color property will use an RGBa or HSLa value. Here, if a browser understand the RGBa or HSLa value it will render it, and if it doesn't, it will fall back to the hexadecimal value above it.

div {
background–color: #b2b2b2;
background–color: rgba(0, 0, 0, .1);
}

Background image

Besides using a background color, it is also possible to use an image as a background via the background–image property, this property allows to control how the image is going to be displayed. Also, it is possible to use the background–image or the shorthand property outright No matter which property we use, there must be an image source identified using a url() function, here, the familiar rules for creating hyperlinks paths apply, which must be inside parenthesis and quoted..

div {
background–image: url("///Users/Guest/Pictures/desert.jpg");
height: 500px; witdh: 100%; }

Notice that only a portion of the image appears, this occur because the set dimensions for the <div> element do not make reference to the image's size, but to the element itself. Additionally other effect that isn't noticeable in the image due to its oversize is the fact that by default the background image will repeat itself horizontally and vertically from the top left of the given element to fill up the element's background. To modify this behaviour the background–repeat and the background–position properties can be used.

Background repeat

As previously mentioned, by default an image will repeat itself both, horizontally and vertically, unless otherwise specified. With the background–repeat it is possible to control how an image repeats itself, if repeated at all; this propertya accepts four different keyword values: repeat, repeat–x, repeat–y, and no–repeat. The first one (repeat) is the default, repeating the image horizontally and vertically.

div {
background-image: url("///Users/Guest/Pictures/airplane.jpg");
}

On the other hand, as its name suggest it the repeat–x, and the repeat–y repeat the background horizontally and vertically respectively. Lastly, the no–repeat prevents the background image from being displayed more than once—that is, do not repeat it at all.

div {
background–image: url("///Users/Guest/Pictures/airplane.jpg");
background–repeat: no–repeat;
height: 500px;
width: 100%;
}

Background position

By default, images are to be displayed from the element's top left corner, anyway, this condition can be modified too using the background-position. This property requires to values: a horizontal offset (the first value), and a vertical offset (second value), each offset is measured from the top left corner, hence, a background-position: 100px 25% will move the image's top left corner 100px to the right, and to the 35% of the element's height. Any type of length value can be used for this property, commonly, as they offer precise control over where the background will be positioned, pixels are oftenly used.

div {
background–image: url("///Users/Guest/Pictures/airplane.jpg");
background–position: 100px 25%;
height: 500px;
width: 100%;
}

Shorthand background images values

The background–color, background–image, background–position, and background–repeat properties may be rolled up into a shorthand value for the background property alone. The order of these properties as a shorthand background property value may vary, how ever it commonly falls as background–color, background–image, background–position, and then background–repeat.

div {
background: #b2b2b2 url("///Users/Guest/Pictures/airplane.jpg") 20px 10px no-repeat;
}

Gradients

In addition to background colors and pictures, another option —introduced in CSS3— is to use gradients, previously, if a gradient were to be used, this one had to be created with image-processing software, besides the extra effort, and the process to recut and image to upload it to a server was very inflexible. Now, it is possible not only to select two or more colors for a gradient, but also modify its orientation, and select it as linear or radial.

Gradient background vendor prefixes

Because of its relatively new introduction, and in order to guarantee fully support in recent browser versions (backwards compatibility is not applicable) when working with gradients is useful to add vendor prefixes.

div {
background: #466368;
background: –webkit–linear–gradient(#648880, #293f50);
background: –moz–linear–gradient(#648880, #293f50);
background: linear–gradient(#648880, #293f50);
}

Linear gradients

The linear gradients are set using the linear–gradient() function inside a background or background–image property. This function must include two color values, the first os which will be the beginning color value and the second of which will be the ending color value. The browser will then handle the transition between the two colors. Before any gradient backgrounds are identified, we'll also put in a default background property with a solid color. The solid color is to be used as a fallback should a browser not support gradient backgrounds.

Changing the direction of a gradient background

When setting up a gradient, the browser will display the transition from top to bottom. This direction, however, may be changed with the use of keywords or a degree value stated before any color values. For example, should a gradient move from left to the right of an element, in this case, it is possible to use the keyword value to right to identify the direction in which the linear gradient should progress. Also, keyword values may be combined. If a gradient move from the left top to the right bottom of an element, the to right bottom keyword should be used.

div {
background: #466368;
background: linear–gradient(to right bottom, #648880,#293f50);
}

When a diagonal gradient is used in an non–squared element, the transition will not proceed from one corner to another, instead, the gradient will identify the absolute center of the element, place anchors in the perpendicular corners from where it should progress, and then move to the general direction of the corner stated within the value. These corners the gradient moves towards are called "magic corners", as they are not absolute. More information can be gotten at Eric Mayer's article Linear Gradient Keywords.

Besides keywords, it is also possible to use degrees values, which is measure from a vertical line in a clockwise direction. For instance, to move a gradient to the left top of an element, a value of 315deg must be used, or alternatively, to move a gradient to the bottom right, a value of 135deg can be used. This same concept can be applied for any degree value from 0deg through 360deg.

.ex-gradient-04 {
background: linear-gradient(222deg, #fff, #293f50);
background-color: teal;
}

Radial gradient background

In the same manner it is possible to work with linear gradients, with the radial–gradient() is possible to make a transition with two or more colors in a circular fashion. This function will sit in the element's absolute center, displaying the first color inside and the second color in the outter part. Opposite to linear gradients, radial gradients can get quite complex, with values for location, size, radius, and so forth. This link provide further information in this regard.

Gradient color stops

At a minimun, gradient backgrounds will transition from one color to another, however, as it is possible to work with more colors, in such cases, is necessary to add color stops to indicate how the transitions between them should proceed. To do this these stops are added to the given gradient function by means of commas, separating each color stop from the next.

By default, the transition between multiple colors will be equally spaced, but if more control over how colors are positione is desired, a location along the gradient may be identified for each color stop. The location should be declared as a length values and should fall after the color value. Is necessary to state that unless declared otherwise, the first and last color will be positioned at a 0% and 100% length values.

div {
background: #648880;
background: linear–gradient(to right; #f6f1d3, #648880, #293f50);
}
div {
background: #648880;
background: linear–gradient(to right, #f6f1d3, #648880 85%, #293f50);
}

Multiple background images

Just until the introduction of CSS3 was possible to set up multiple background images by comma separated values into the background or background–image property. This made easier work around some constrains when designing a page.

The background image value that comes first will work as the foremost background image, whilst the last background image will be the rearmost background image. Any value in between will reside within the middle ground accordingly.

div {
background: url("foreground.png") 0 0 no–repeat, url("middle–ground.jpg") 0 0 no–repeat, url("background.jpeg") 0 0 no–repeat;
}
